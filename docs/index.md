## How changes are handled

All documentation updates follow a **Docs-as-Code workflow**:

1. Create a branch and update Markdown files.
2. Open a Pull Request (PR).
3. Automated checks run:
   - Style lint (Vale)
   - Build validation (MkDocs)
   - AI review (Deviation Alert report)
4. A human reviewer approves or requests changes.
5. After approval, the PR is merged and the site is published.
